    # RabbitMQ基础

    # 1. MQ的基本概念

    优势：

    • 应用解耦：提高系统容错性和可维护性
    • 异步提速：提升用户体验和系统吞吐量
    • 削峰填谷：提高系统稳定性

    劣势：

    • 系统可用性降低
    • 系统复杂度提高
    • 一致性问题

    # 1.1 优势

    应用解耦：

    ​ 未使用MQ之前的执行速度

    ​ 未使用MQ时，高峰流量会导致系统崩溃

    ​ 使用MQ后，可以限制每秒处理多少请求，将流量峰值时间段推迟到空闲时间段继续请求

    # 2. RabbitMQ简介

    ​ 2007年，Rabbit技术公司基于AMOP标准开发的RabbitMQ 1.0发布。RabbitMQ采用Erlang语言开发。Erlang语言由Ericson设计，专门为开发高并发和分布式系统的一种语言，在电信领域使用广泛。

    ​ RabbitMQ基本架构如图

    RabbitMQ提供了6种工作模式：简单模式、work queues、Publish/Subscribe发布与订阅模式、Routing路由模式、Topics主题模式、RPC远程调用模式（不太算MQ；暂不介绍）

    # 小结

    1. RabbitMQ是基于AMQP协议使用Erlang语言开发的一款消息队列产品。
    2. RabbitMQ提供了6种工作模式。
    3. AMQP是协议，类比HTTP
