    # ElasticSearch基础讲解与安装

    # 1、为什么使用ElasticSearch ？

    # 案例场景：

    ​ 需要模糊查询一条数据，数据量有千万级。

    # 实现方法：

    1. 使用SQL语句的like查询？

      select * from goods where title like '%荣耀%'; 数据量庞大时，应该如何处理？ 商品数据量超过千万，一次查询得花费大量时间。

    2. 使用全文检索（ElasticSearch等）

      主流搜索技术

    # 2、什么是全文检索

    1. # 数据的分类

      数据总共分为两类：结构化数据、非结构化数据

      # 结构化数据：

      ​ 固定格式、有限长度、数据类型固定

      ​ 例如：数据库中的数据就是结构化数据

      # 非结构化数据：

      ​ 格式不固定、长度不固定、数据类型不固定

      ​ 例如：磁盘上的文件

    2. # 结构化数据的搜索（数据库）

      直接使用SQL语句进行查询

    3. # 非结构化数据的查询

      需求：找出文件内容中包含Java的所有文件

      # 实现思路：

      1. # 顺序扫描---暴力模式

        数据量庞大，性能差。

      2. # 把非结构化数据变成结构化数据

        基于结构化数据创建索引，然后在查询。

        索引：是一个独立的数据结构，b+tree结构。为了提高查询效率创建，需要独立占用磁盘空间。

        停用词：没有意义的词，例如英文中的and、the、a、an等。

        全文检索：先对内容进行分词处理，根据空格对文章内容进行字符串拆分。合并重复的单词、去除标点符号、去除停用词。 创建索引，然后查询索引的过程就叫做全文检索。 索引可以一次创建多次使用，表现为每次查询速度都非常快。

    # 3、全文检索的应用场景

    1. # 搜索引擎

      使用爬虫程序抓取网页的内容，分词创建索引。

      搜索时，直接查询索引。

    2. # 站内搜索

      电商搜索 论坛搜索 微博搜索

    3. # 搜索磁盘上的文件

    4. # 只要有查询的地方就可以使用全文检索。

    # 4、如何实现全文检索

    1. # 使用lucene，在java中唯一的一个全文检索技术。

      一个全文检索工具包。很少直接使用。相当于ES和Solr等技术的底层工具包

    2. # Solr、ElasticSearch（ES）

      推荐使用。 都是基于Lucene的全文检索服务器。独立运行，支持集群。 实时搜索时，ES的性能更好。

    # 5、全文检索的流程 (仅讲述流程,未有实例)

    1. # 创建索引

      1. # 采集数据

        可以各种新式的数据源 搜索引擎：爬虫，从互联网抓取数据 站内搜索：数据库 文件搜索：磁盘上的文件

      2. # 创建文档对象

        把原始文档进行封装。Document对象。 其中包含多个field 每个field可以自定一个名称，不同的Document中可以有不同的field。

      3. # 分析文档

      对其中需要分析的field进行分析处理，可以在field中进行指定。 分词：进行关键词拆分。 去除标点符号，合并相同的单词。 去除停用词 转换大小写 最终得到一个单词列表

      其中并不是所有的field都需要分析，例如文件的路径。 单词列表中，其中一个关键词就是一个term。

      1. # 创建索引

        其中包含三部分内容： 1）索引：基于关键词（term）创建的。 2）文档对象：Document，第二步封装好的Document对象。 3）关键词和文档的对应关系

    2. # 查询索引

      1. # 用户接口

        用户输入查询内容 输入可以是一个关键词，也可以是一句话。

      2. # 封装查询条件

        需要把用户输入的内容进行分词处理。得到一个单词列表。 需要指定要查询的field。 查询条件： field：value

      3. # 执行查询

        在索引中根据查询条件查询对应的字段，找到对应的关键词。 如果关键词在索引中存在的就能查询到结果。如果不存在就没有结果。 根据关键词找到对应的文档id列表，根据文档id查询到对应的Document对象。

      4. # 展示结果

        关键词的高亮显示，分页处理，根据相关度进行排序。
